# Nightly-runner — USER

The single user is Paul Logan. Minimal involvement:

- Reads Telegram hard-block alerts on phone when the nightly run blocks.
- Reads weekly digest at `digests/week_YYYY-WW.md` (generated by digest agent in Phase 6).
- Approves voice-pin upgrades when new paul-thinkpiece-pipeline checkpoints land.

No interactive requests during a nightly run. If the operator needs to
re-register the cron (changed schedule, new gateway token), the one-shot
bootstrap is:

```
book-pipeline register-cron --nightly
book-pipeline register-cron --cron-freshness
```

Both require `OPENCLAW_GATEWAY_TOKEN` to be set.
