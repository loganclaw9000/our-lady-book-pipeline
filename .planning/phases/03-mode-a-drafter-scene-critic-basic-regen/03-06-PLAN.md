```jinja2
{#- regen.j2 — Scene-local regenerator prompt.
    Variables:
      prior_scene_text: str
      severity_grouped_issues: dict[str, list[CriticIssue]]  # high, mid, low (low = context only)
      word_count_target: int  # ±10% of original
      scene_request: SceneRequest
      voice_description: str
      retrievals: dict[str, RetrievalResult]
-#}
===SYSTEM===
You are rewriting specific passages of a scene to fix critic issues. Voice: {{ voice_description }}

Rules:
1. Target total word count: {{ word_count_target }} (±10%). Drifting outside this band fails the rewrite.
2. Address every HIGH and MID severity issue listed. LOW-severity issues are context; do not chase them if it costs word budget.
3. Preserve all named entities, dates, and facts from the corpus context below.
4. Preserve passages NOT implicated by an issue — minimize change outside affected ranges.

Corpus context (same context the drafter saw):
{% for retriever_name, result in retrievals.items() -%}
--- {{ retriever_name }} ---
{% for hit in result.hits -%}
[{{ hit.chunk_id }}] {{ hit.text }}
{% endfor %}
{% endfor %}

===USER===
Scene: ch{{ "%02d" | format(scene_request.chapter) }}_sc{{ "%02d" | format(scene_request.scene_index) }}
POV: {{ scene_request.pov }} | Date: {{ scene_request.date_iso }} | Location: {{ scene_request.location }}

Prior scene (BEFORE rewrite):
'''
{{ prior_scene_text }}
'''

Issues to fix (HIGH first):
{% for severity in ["high", "mid"] -%}
{% for issue in severity_grouped_issues.get(severity, []) -%}
- [{{ severity }}/{{ issue.axis }}] {{ issue.location }}: {{ issue.claim }}
  Evidence: {{ issue.evidence }}
  {% if issue.citation %}Citation: {{ issue.citation }}{% endif %}
{% endfor %}
{% endfor %}

{% if severity_grouped_issues.get("low") -%}
Low-severity context (don't chase):
{% for issue in severity_grouped_issues["low"] -%}
- [{{ issue.axis }}] {{ issue.claim }}
{% endfor %}
{% endif %}

Rewrite the scene. Return ONLY the full revised scene text. No meta commentary. No outline prefix.
```

Regen strategy note: because targeted splicing of char ranges in a narrative is fragile (sentence boundaries drift, prose doesn't slot back cleanly), the pragmatic implementation asks Opus for the FULL revised scene with explicit instructions to minimize change outside affected ranges. Word-count guard enforces ±10%. This matches CONTEXT.md decision "Preserve exactly [unaffected passages quoted]".

Word-count guard (post-process): prior_word_count = len(prior.scene_text.split()); new_word_count = len(new_scene_text.split()). If abs(new - prior) / prior > 0.10: emit error Event + raise RegenWordCountDrift. Plan 03-07 scene loop catches and transitions back to CRITIC_FAIL (attempt counts toward R).
