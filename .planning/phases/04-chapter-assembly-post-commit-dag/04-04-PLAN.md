<objective>
Wire the 4 Plan 04-02 + 04-03 concretes (ConcatAssembler + ChapterCritic + OpusEntityExtractor + OpusRetrospectiveWriter) into a single orchestrator that drives the strict 4-commit post-commit DAG:

1. **Canon commit** (`canon/chapter_{NN}.md`): ConcatAssembler runs → ChapterCritic reviews fresh pack → on PASS, write markdown, `git add canon/chapter_{NN}.md`, `git commit -m "canon(ch{NN}): commit <title>"`. Capture `chapter_sha = git rev-parse HEAD` for stale-card detection. On FAIL → state → CHAPTER_FAIL, no commit, terminal state for Phase 4.
2. **Entity-extraction commit** (`entity-state/chapter_{NN:02d}_entities.json`): load all prior chapter entity JSONs, flatten to list[EntityCard], call `OpusEntityExtractor.extract(chapter_text, NN, chapter_sha, prior_cards)`, write JSON, commit `chore(entity-state): ch{NN} extraction`. Failure → DAG_BLOCKED.
3. **RAG reindex commit**: rebuild `entity_state` LanceDB table FULLY from `entity-state/chapter_*.json` (idempotent, ≤500 rows — per CONTEXT.md grey-area d); call `ArcPositionRetriever.reindex()` (Plan 02-04 Protocol-conformant no-arg). Commit the lancedb file changes under `indexes/` with message `chore(rag): reindex after ch{NN}`. (Even though `indexes/*.lance/` is `.gitignored`, the commit message is the auditable handle; commit wraps `indexes/resolved_model_revision.json` which IS tracked.) Failure → DAG_BLOCKED.
4. **Retrospective commit** (`retrospectives/chapter_{NN:02d}.md`): call `OpusRetrospectiveWriter.write(chapter_text, chapter_events, prior_retros)`, write markdown with YAML frontmatter, commit `docs(retro): ch{NN}`. Ungated per CONTEXT.md — failure logged, DAG still advances to DAG_COMPLETE (write a stub retrospective so the path exists).

Post-completion: on DAG_COMPLETE, archive `drafts/scene_buffer/ch{NN}/*` to `drafts/scene_buffer/archive/ch{NN}/` via git mv; clear `drafts/ch{NN}/*.md` (scenes are now in canon). Write `.planning/pipeline_state.json` with the `last_committed_chapter, last_committed_dag_step=4, dag_complete=True` view.

Parallel: **AblationRun harness skeleton** (TEST-01 ablation side) — Pydantic dataclass + directory skeleton. Phase 6 fills in actual A/B execution; Phase 4 ships only the on-disk shape so Phase 6 doesn't retrofit.

Purpose: This is the core LOOP-02 + LOOP-03 wiring. Strict ordering enables per-step rollback + resumability. The `dag_step` integer on `ChapterStateRecord` is load-bearing: re-running `book-pipeline chapter <N>` on a partial run resumes at the failed step.

Output:
- `ChapterDagOrchestrator.run(chapter_num)` function returns a `ChapterStateRecord` (terminal state ∈ {DAG_COMPLETE, CHAPTER_FAIL, DAG_BLOCKED}).
- 4-commit sequence verified by integration test with mocked LLM clients.
- Resumability verified by running orchestrator, killing after step 2, re-running → steps 1+2 skipped, 3+4 complete.
- `AblationRun` pydantic + `create_ablation_run_skeleton` function.
</objective>

       """AblationRun config + on-disk skeleton (TEST-01 foundation).

       Phase 4 ships the config shape + directory layout so Phase 6 TEST-03 has
       an established on-disk contract to build against. This module contains
       NO execution logic; actual variant-A vs variant-B runs land in Phase 6.
       """
       from __future__ import annotations
       import json
       from datetime import UTC, datetime
       from pathlib import Path
       from typing import Literal
       from pydantic import BaseModel, Field


       class AblationRun(BaseModel):
           """Pydantic config for one ablation run.

           Carries SHA pins for every dimension of reproducibility required by
           TEST-03 (config, corpus, checkpoint). `run_id` is typically an ISO
           timestamp with microsecond precision — caller supplies; harness does
           not generate.
           """
           run_id: str
           variant_a_config_sha: str
           variant_b_config_sha: str
           n_scenes: int = Field(ge=1)
           corpus_sha: str
           voice_pin_sha: str
           created_at: str
           status: Literal["pending", "running", "complete", "failed"] = "pending"
           notes: str = ""


       def create_ablation_run_skeleton(
           run: AblationRun,
           ablations_root: Path = Path("runs/ablations"),
       ) -> Path:
           """Create `{ablations_root}/{run.run_id}/{a,b}/` + `ablation_config.json`.

           Idempotent: if the run dir exists, the config JSON is rewritten but
           variant subdirectories are not emptied. Returns the run root path.
           """
           run_dir = ablations_root / run.run_id
           run_dir.mkdir(parents=True, exist_ok=True)
           (run_dir / "a").mkdir(exist_ok=True)
           (run_dir / "b").mkdir(exist_ok=True)
           (run_dir / "a" / ".gitkeep").touch(exist_ok=True)
           (run_dir / "b" / ".gitkeep").touch(exist_ok=True)
           config_path = run_dir / "ablation_config.json"
           # Atomic: tmp + rename (matches orchestrator state persistence pattern).
           tmp = config_path.with_suffix(".json.tmp")
           tmp.write_text(run.model_dump_json(indent=2), encoding="utf-8")
           tmp.replace(config_path)
           return run_dir


       def utc_timestamp() -> str:
           """Return a microsecond-precision UTC ISO-8601 timestamp usable as a run_id prefix."""
           return datetime.now(UTC).isoformat(timespec="microseconds").replace("+00:00", "Z")


       __all__ = ["AblationRun", "create_ablation_run_skeleton", "utc_timestamp"]
