<objective>
Land two kernel concretes that convert committed scenes into a chapter-critic-ready artifact:

1. **ConcatAssembler** — the deterministic, no-LLM scene-join that satisfies the frozen `ChapterAssembler` Protocol. Reads `drafts/ch{NN}/*.md` (scene markdown with YAML frontmatter, per B-3 invariant from Plan 03-07), sorts by scene_index, concatenates with `\n\n---\n\n` section breaks, wraps each scene in an HTML comment `<!-- scene: ch{NN}_sc{NN} -->` for Phase 6 traceability, aggregates scene frontmatter into a chapter-level frontmatter dict, and returns the joined chapter markdown string.

2. **ChapterCritic** — the second `Critic` Protocol impl (level=`chapter`, rubric_id=`chapter.v1`). Mirrors `SceneCritic` structure (pre-rendered cached system prompt + messages.parse via claude_code_cli backend + CRIT-04 audit log + rubric_version stamping + tenacity retry + one Event per invocation) BUT differs in three critical ways: (a) stricter pass threshold (≥3/5 per axis, not ≥70/100 — use rubric_version='chapter.v1' thresholds); (b) builds its OWN fresh ContextPack via caller injection (breaks C-4 drafter/critic collusion); (c) audit dir is `runs/critic_audit/` with filename prefix `chapter_{NN:02d}_{timestamp}.json` (vs scene's `{scene_id}_{attempt:02d}_...`).

Purpose: Satisfies CRIT-02 + LOOP-02 success criteria 1 + 2 (deterministic assembly + chapter critic with independent pack). The fresh-pack rule is the CORE mitigation for PITFALLS C-4 (drafter + critic sharing the same pack silently produces inflated pass rates). Test asserts `chapter_pack.fingerprint not in {scene_pack.fingerprint for scene in chapter}`.

Output:
- `book_pipeline.chapter_assembler.concat.ConcatAssembler` class + instance satisfies `isinstance(a, ChapterAssembler)` runtime check.
- `book_pipeline.critic.chapter.ChapterCritic` class satisfies `isinstance(c, Critic)` and `c.level == 'chapter'`.
- `config/rubric.yaml` has a new `chapter_v1` block (rubric_version=`chapter.v1`, per-axis thresholds keyed on the ≥3/5 rule).
- Two template assets: `chapter_system.j2` + `chapter_fewshot.yaml` (1 bad + 1 good example).
- Unit tests: 6 for ConcatAssembler (frontmatter parse, scene sort, HTML comment markers, deterministic re-run, edge cases) + 10 for ChapterCritic (Protocol conformance, fresh pack invariant, ≥3/5 threshold, rubric_version stamp, audit log success + failure, Event emission, tenacity exhaustion, cache-control list identity, missing-axes fill, invariant fix).
