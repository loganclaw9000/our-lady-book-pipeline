3. `src/book_pipeline/rag/chunker.py`:
   - `chunk_markdown(text: str, source_file: str, *, target_tokens: int = 512, overlap_tokens: int = 64, ingestion_run_id: str = "") -> list[Chunk]`.
   - Tokenization: use tiktoken `cl100k_base` encoding (already in pyproject) for token counts. Not perfect for BGE-M3 but within ±15% is fine per PITFALLS R-4 mitigation (heading boundaries matter more than exact token count).
   - Algorithm: split text on `^#{1,6} ` headings (regex, multiline). Build heading_path as "H1 > H2 > H3" joined-breadcrumb of all enclosing headings. Within each heading section, sliding-window over sentences aggregated to target_tokens with overlap_tokens carry-over. DO NOT cross heading boundaries in a single chunk.
   - rule_type inference: case-insensitive regex on the leaf heading: if matches `hypothetic` → "hypothetical"; `\bexample(s)?\b` → "example"; `cross[- ]?ref` → "cross_reference"; else "rule".
   - chapter inference: scan the heading_path for a top-level `Chapter N` heading via regex `^#\s+Chapter\s+(\d+)` OR breadcrumb segment matching `Chapter\s+(\d+)`. If found, `chunk.chapter = int(N)`; otherwise `chunk.chapter = None`. Per W-5: arc_position retriever filters on this column directly (exact equality) instead of the fragile `heading_path LIKE 'Chapter N %'` approach.
   - chunk_id: `from book_pipeline.observability.hashing import hash_text; chunk_id = hash_text(f"{source_file}|{heading_path}|{chunk_text}")`.
   - Empty/whitespace-only sections produce 0 chunks (no empty-text Chunk).

5. `src/book_pipeline/rag/lance_schema.py`:
   ```python
   import pyarrow as pa
   from pathlib import Path
   import lancedb
   from book_pipeline.rag.embedding import EMBEDDING_DIM

   # Per W-5 (revision): `chapter` column added at schema-creation time so
   # arc_position retriever (Plan 04) can filter by exact chapter equality
   # rather than fragile `heading_path LIKE 'Chapter N %'` matching. `chapter`
   # is nullable because only outline.md / brief.md chunks have chapter context;
   # glossary/maps/engineering rule cards do not. Additive-only schema change —
   # no existing field renamed or reordered.
   CHUNK_SCHEMA = pa.schema([
       pa.field("chunk_id", pa.string(), nullable=False),
       pa.field("text", pa.string(), nullable=False),
       pa.field("source_file", pa.string(), nullable=False),
       pa.field("heading_path", pa.string(), nullable=False),
       pa.field("rule_type", pa.string(), nullable=False),
       pa.field("ingestion_run_id", pa.string(), nullable=False),
       pa.field("chapter", pa.int64(), nullable=True),
       pa.field("embedding", pa.list_(pa.float32(), EMBEDDING_DIM), nullable=False),
   ])

   def open_or_create_table(db_path: Path, axis_name: str) -> "lancedb.table.Table":
       """Open the lance table for axis_name at db_path, creating empty with CHUNK_SCHEMA if absent.
       If the table exists with a different schema, raises RuntimeError (never silently migrates)."""
       db_path.mkdir(parents=True, exist_ok=True)
       db = lancedb.connect(str(db_path))
       if axis_name in db.table_names():
           tbl = db.open_table(axis_name)
           if tbl.schema != CHUNK_SCHEMA:
               raise RuntimeError(f"Schema mismatch on table {axis_name!r}: expected CHUNK_SCHEMA, got {tbl.schema}")
           return tbl
       return db.create_table(axis_name, schema=CHUNK_SCHEMA, mode="create")
   ```
