scripts/lint_imports.sh (CI-friendly wrapper — mypy scope matches per-plan gates):

```bash
#!/usr/bin/env bash
# Single-command boundary + lint check. Runs import-linter + ruff + mypy.
# Exits non-zero if any check fails.
#
# IMPORTANT — mypy scope:
#   Plans 01-05 each ran `uv run mypy <specific-package>` as their per-plan
#   gate (e.g. plan 02: mypy src/book_pipeline/interfaces src/book_pipeline/stubs;
#   plan 03: mypy src/book_pipeline/config; plan 05: mypy src/book_pipeline/observability).
#   A whole-tree `uv run mypy src` here would sometimes fail due to cross-module
#   inference that didn't surface in per-plan checks, making this aggregate gate
#   stricter than the per-plan gates — a confusing last-mile failure.
#
#   Fix: scope this aggregate mypy step to the EXACT set of packages Phase 1
#   plans touched. When Phase 2+ add new kernel packages (rag/, drafter/, ...),
#   append them here in the same PR that creates them — each phase owns the
#   extension of this list.
set -euo pipefail
cd "$(dirname "$0")/.."

echo "[1/3] import-linter..."
uv run lint-imports

echo "[2/3] ruff check..."
uv run ruff check src tests

echo "[3/3] mypy (Phase 1 packages)..."
# One mypy invocation listing all Phase 1 packages — matches per-plan scopes.
# Each listed target corresponds to a plan's mypy gate:
#   - interfaces + stubs     → plan 02
#   - config                 → plan 03
#   - openclaw               → plan 04
#   - observability          → plan 05
#   - cli                    → plans 01/03/04/05 (entry points)
#   - book_specifics         → plan 06 (this plan)
# Phase 2 adds: src/book_pipeline/rag
# Phase 3 adds: src/book_pipeline/drafter, src/book_pipeline/regenerator
# Phase 4 adds: src/book_pipeline/critic, src/book_pipeline/entities, src/book_pipeline/retrospectives
# Phase 5 adds: src/book_pipeline/orchestration
uv run mypy \
  src/book_pipeline/interfaces \
  src/book_pipeline/stubs \
  src/book_pipeline/observability \
  src/book_pipeline/config \
  src/book_pipeline/openclaw \
  src/book_pipeline/cli \
  src/book_pipeline/book_specifics

echo "OK — module boundaries, ruff, and scoped mypy all pass."
```

    This file should NEVER exist outside the scope of that test — if you see it in
    a commit, delete it. It exists only to prove import-linter catches a kernel ->

    \"\"\"
    from book_pipeline.book_specifics import corpus_paths  # noqa: F401  — intentional violation
